## Color & Style Conventions

**Never use raw `Colors.*` constants in widget code.** Always reference:

```dart
AppColors.primaryColor     // Color(255, 178, 0) — amber/gold brand color
AppColors.backgroundColor  // Color(224, 248, 250) — icy blue scaffold background
AppColors.surfaceColor     // Colors.white — cards, sheets, dialogs
AppColors.textPrimary      // black87
AppColors.textSecondary    // black54
AppColors.textHint         // black38
AppColors.divider          // 0xFFE0E0E0
AppColors.error            // Colors.redAccent
```

**Text styles** — always use `AppTextStyles.*TextStyle`:

```dart
AppTextStyles.titleTextStyle                // Section headers (20, bold)
AppTextStyles.currencyItemTitleTextStyle    // Currency ticker (16, bold)
AppTextStyles.currencyItemSubtitleTextStyle // Currency subtitle (12, normal)
AppTextStyles.infoLabelTextStyle            // Info row label (13, w500)
AppTextStyles.infoValueTextStyle            // Info row value (13, w600)
AppTextStyles.fieldLabelTextStyle           // TextField floating label
AppTextStyles.fieldHintTextStyle            // TextField hint
```

**Padding** — use `EdgeInsets.all(20)` for cards, `EdgeInsets.symmetric(horizontal: 20, vertical: 12)` for list items.

## State Management

`CalculatorCubit` is the single cubit. States:

```
CalculatorInitial   (startup)
CalculatorLoading   (API call in flight)
CalculatorLoaded    { exchangeRate: double, convertedAmount: double }
CalculatorError     { message: String }
```

- The cubit is **registered as a factory** in GetIt — fresh instance per `BlocProvider`.
- `BlocProvider` is created in `CalculatorScreen`, not in `main()`.
- Use `BlocBuilder` for rebuilding UI on state, `BlocListener` for side effects (dialogs).

## Exchange Service Strategy

The official API endpoint (`/orderbook/public/recommendations`) requires
**internal numeric asset IDs** that are not publicly exposed — catalog routes
return 404. Until those IDs are known:

- `AppConfig.useMockExchange = true` (compile-time, via `--dart-define=USE_MOCK_EXCHANGE=false` to override)
- **Active:** `MockExchangeService` → CoinGecko free API, no key required
- **Blocked:** `RealExchangeService` → stub ID maps with `// ⚠ placeholder` comments

To add a new crypto asset to the mock service, add an entry to
`MockExchangeService._cryptoIdMap` (key = `Currency.apiId`, value = CoinGecko slug).

## Currency Data

Currency lists are **static constants** on `ConvertionBottomSheet`:

```dart
ConvertionBottomSheet.fiatCurrencies    // BRL, COP, PEN, VES
ConvertionBottomSheet.cryptoCurrencies  // USDT (TATUM-TRON-USDT), USDC (TATUM-TRON-USDC)
```

The `Currency` model lives in `lib/data/models/currency.dart`. The `apiId`
field is what gets sent to the exchange service — it may differ from the
display `id` (e.g. `id: 'USDT'`, `apiId: 'TATUM-TRON-USDT'`).

## Conversion Direction Logic

`_isFiatFirst` (bool, owned by `_CalculatorWidgetState`) controls direction:

| `_isFiatFirst` | API `type` | Tengo | Quiero | Formula |
|---|---|---|---|---|
| `true` | `1` | fiat | crypto | `convertedAmount = amount × exchangeRate` |
| `false` | `0` | crypto | fiat | `convertedAmount = amount ÷ exchangeRate` |

`exchangeRate` from the API always means **crypto units per 1 fiat unit**.

## Key Gotchas

1. **`Colors.amber` is banned** — always use `AppColors.primaryColor`.
2. **`CurrencyIcon`** is in `common/widgets/` — do not re-create an inline version.
3. **`_InfoSection` was extracted** to `common/widgets/info_section.dart` as `InfoSection` (public).
4. **`Currency` model** is in `data/models/currency.dart`, NOT in `convertion_bottomsheet.dart`.
5. The **mock file** (`*.mocks.dart`) is generated — running `build_runner` will recreate it.
6. `CalculatorScreen` reads `screenHeight` at build time for the decorative sphere positioning — do not use a fixed size.
7. `floatingLabelBehavior: FloatingLabelBehavior.always` is set on all TextFields so labels always float above.
